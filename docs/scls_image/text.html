Generate simple text

Separate datas and images

To be clearly rendered, a text needs a lot of meta datas : color, size... However, generating text is a quite heavy process, which needed a lot of ressources. To not require too much ressources, the meta datas and images are separated. Indeed, we can only manage to edit the datas precisely, without doing heavy modification on the image.

Generate a block of text

The lowest-level text you can generate is called a block of text. A block of text is a text which the exactly same outline style in all the block. It takes UTF-8 text as an input, but converts it to code point for internal use.

Generating a single block remains a quite complicated process. The first step is to parse datas of the block (text, style...). This is the faster step, and can be done every time a little modification is applied to the text. In Text_Image_Block, this process is made by the update_line_text() method.
The next step is the generation of the text. Actually, the text is generated line by line. Indeed, if a single line of a big text is modified, only this line will be regenerated. This can be checked with _check_modified_lines(). The generation of the singles lines is mixed between a lot of methods. The method generate_lines() calls the generation of all the lines. This request is send to _regenerate_lines(), which will sort the already existing lines, to use the usables one and remove the useless ones. More precisely, the useless lines are removed with delete_useless_generated_lines(). An another method, clear_lines(), clear the memory from all the contained lines. The line is created and rendered in generate_next_line(). However, the Text_Image_Line object is created in _generate_line().
